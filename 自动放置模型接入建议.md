# 自动放置模型接入建议（三档）

本文给出三档可落地路线：规则增强、轻量重排、学习策略。每档包含下载建议、输入输出约定、在当前脚本中的最小改动点。

## 1) 规则增强（零训练，立即可用）

适用场景：先把稳定性和可解释性做起来，快速降低重叠率与失败率。

下载建议：
- 不依赖额外大模型。
- 可选下载公开室内先验词典（对象-房间共现统计），存成本地 JSON。

输入输出：
- 输入：候选点、房间 bbox、对象关键词画像。
- 输出：候选分数（越大越好）。

最小改动点（已在脚本预留）：
- 候选打分入口：sample_and_place_objects.py 的 _score_candidate_by_backend。
- 规则实现：sample_and_place_objects.py 的 _score_candidate_rule。
- 使用方式：--placement auto --placement-backend rule。

## 2) 轻量重排（LLM/小模型 reranker）

适用场景：在规则基础上提升语义合理性，例如“时钟更靠墙”、“相机更靠桌面区域”。

下载建议：
- 开源 reranker 或小型指令模型（7B~14B）量化版。
- 若显存受限，优先 GGUF/INT4 版本，离线部署到本机或局域网服务。

输入输出：
- 输入：object_name、room_type、候选点集合（Top-K）、每个候选的几何特征。
- 输出：每个候选的重排分数或最佳候选索引。

最小改动点：
- 在 sample_and_place_objects.py 的 _score_candidate_by_backend 中替换 rerank 分支。
- 维持 auto_place_objects 主循环不变，仅替换打分器。
- CLI 直接使用：--placement-backend rerank。

推荐接入协议：
- 请求 JSON：{object, room, candidates:[...]}
- 响应 JSON：{scores:[...]} 或 {best_index:i}

## 3) 学习策略（policy model）

适用场景：追求更高自动化，减少人工修正步数，允许引入训练或蒸馏。

下载建议：
- 关系建模策略（GNN/Transformer）或扩散式布局模型。
- 优先选择有室内场景数据预训练权重的仓库，再做小规模微调。

输入输出：
- 输入：房间几何、已放置物体状态、目标对象类别。
- 输出：直接生成位姿候选（position/rotation）或候选分布参数。

最小改动点：
- 在 sample_and_place_objects.py 的 _score_candidate_by_backend 增加 policy 分支实现。
- 如需直接生成候选，可新增函数 policy_propose_candidates，再在 auto_place_objects 内替换候选生成环节。
- CLI 直接使用：--placement-backend policy。

## 推荐落地顺序

1. 先用 rule 跑通统计闭环（失败率、重叠率、人工调整步数）。
2. 在 rerank 分支接入服务，保持回退到 rule（服务异常不影响主流程）。
3. 最后引入 policy，先做离线评估，再灰度上线。

## 与当前脚本对应关系

- 主流程入口：sample_and_place_objects.py 的 interactive_sampling_loop。
- 自动放置执行：sample_and_place_objects.py 的 auto_place_objects。
- 候选打分统一钩子：sample_and_place_objects.py 的 _score_candidate_by_backend。
- 失败对象编辑标记：layout 输出字段 editor_focus_failed_ids、editor_review_order，以及对象字段 edit_status/edit_priority/edit_note。

## 示例命令

```bash
python sample_and_place_objects.py \
  --scene 00808-y9hTuugGdiq \
  --mode load \
  --placement auto \
  --placement-backend rule \
  --placement-attempts 32 \
  --placement-seed 42
```

后续切换 rerank/policy 只改一个参数，主流程保持不变。
